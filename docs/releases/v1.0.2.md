# Release v1.0.2 - Auth MVP y Seguridad (OWASP)

Fecha: TBD

Resumen
- Implementación de autenticación completa MVP: registro, login y logout.
- Flujo de sesión con cookies HttpOnly y tokens firmados (OWASP base).
- Conexión frontend con API Next.js para auth (registro/login/logout).
- Tests unitarios para LoginForm/LoginFlow; plan de integración en fases.
- Documentación de la fase Auth en docs/features/33-auth-fullstack.md.

Cambios principales
- Backend: rutas /api/auth/login, /api/auth/register, /api/auth/logout y store en memoria para desarrollo.
- Frontend: flujo de autenticación completo (LoginFlow, RegisterFlow) con rutas de inicio y registro; persistencia de sesión via cookies.
- UI responsive y animada, optimizada para móvil.

Notas de seguridad (OWASP)
- Tokens firmados; cookies HttpOnly; manejo básico de expiración y rotación de tokens.
- Validación de entradas y respuestas genèriques para evitar information leakage.
- CSRF mitigation a través de SameSite y prácticas seguras (en el servidor eventual).

Próximos pasos
- Implementar backend Go real y BD (PocketBase) para auth completo.
- Añadir pruebas de integración end-to-end (Playwright) para login/registro/logout.
- Crear PR a main cuando corresponda y mantener la trazabilidad de cambios.
